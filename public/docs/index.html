Introduction

Base URL: http://localhost

Welcome to the E-Commerce API Documentation.

This API allows you to interact with our e-commerce platform for managing products, orders, users, and more. Whether you're integrating with our product catalog, processing orders, or managing customer information, this documentation will guide you through every available endpoint.

Authenticating requests

To authenticate requests, include an Authorization header with the value "Bearer {YOUR_AUTH_KEY}".

All authenticated endpoints are marked with a requires authentication badge in the documentation below.

You can retrieve your token by visiting your dashboard and clicking Generate API token.

Endpoints

Authentication

Authenticate the user and return their API token.

This endpoint is used to authenticate a user with their email and password. If the credentials are valid, an API token and refresh token will be issued.

Example request:
curl --request POST \
    "http://localhost/api/login" \
    --header "Content-Type: application/json" \
    --header "Accept: application/json" \
    --data "{
    \"email\": \"tankunding@example.net\",
    \"password\": \"dPJUAO<{jO\'Lpfqk<_\"
}"

const url = new URL(
    "http://localhost/api/login"
);

const headers = {
    "Content-Type": "application/json",
    "Accept": "application/json",
};

let body = {
    "email": "tankunding@example.net",
    "password": "dPJUAO<{jO'Lpfqk<_"
};

fetch(url, {
    method: "POST",
    headers,
    body: JSON.stringify(body),
}).then(response => response.json());
$client = new \GuzzleHttp\Client();
$url = 'http://localhost/api/login';
$response = $client->post(
    $url,
    [
        'headers' => [
            'Content-Type' => 'application/json',
            'Accept' => 'application/json',
        ],
        'json' => [
            'email' => 'tankunding@example.net',
            'password' => 'dPJUAO<{jO\'Lpfqk<_',
        ],
    ]
);
$body = $response->getBody();
print_r(json_decode((string) $body));

Example response (200):


{
    "data": {
        "token": "{YOUR_AUTH_KEY}",
        "refresh_token": "{YOUR_REFRESH_KEY}"
    },
    "message": "Authenticated",
    "status": 200
}

Example response (401):


{
    "message": "Invalid credentials",
    "status": 401
}

Register a new user and return their API token.

This endpoint is used to register a new user, including their name, email, password, and password confirmation. A new API token and refresh token will be issued.

Example request:
curl --request POST \
    "http://localhost/api/register" \
    --header "Content-Type: application/json" \
    --header "Accept: application/json" \
    --data "{
    \"name\": \"John Doe\",
    \"email\": \"john.doe@example.com\",
    \"password\": \"password123\",
    \"password_confirmation\": \"password123\"
}"

Logout the user and destroy their API token.

requires authentication

This endpoint allows the user to log out by deleting their current access token and refresh token from the database.

Example request:

    "http://localhost/api/logout" \
    --header "Authorization: Bearer {YOUR_AUTH_KEY}" \
    --header "Content-Type: application/json" \
    --header "Accept: application/json" \
    --data "{
    \"refresh_token\": \"eljnfguppzikqszhrwqimytb\"

Refresh the user's API token and generate a new refresh token.

requires authentication

This endpoint is used to refresh the user's API access token by validating their refresh token and issuing new tokens if valid.

Example request:

    "http://localhost/api/token/refresh" \
    --header "Authorization: Bearer {YOUR_AUTH_KEY}" \
    --header "Content-Type: application/json" \
    --header "Accept: application/json" \
    --data "{
    \"refresh_token\": \"giapvc\"
}"
